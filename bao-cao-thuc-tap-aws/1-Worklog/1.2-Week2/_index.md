### Week 2 objectives:

* Learn AWS Identity and Access Management (IAM) and the principle of least privilege.
* Get familiar with Amazon EC2, virtual servers, AMIs, instance types, key pairs, and Security Groups.
* Practice creating simple resources while keeping costs under control.

### Tasks implemented this week:

| Day | Task | Start date | Completion date | Reference |
| --- | --- | --- | --- | --- |
| Monday | - Review Week 1 content <br> - Learn IAM User, Group, Role, and Policy <br> - Distinguish direct permissions from permissions through roles | 27/04/2026 | 27/04/2026 | IAM |
| Tuesday | - Create an IAM user for lab practice <br> - Configure permissions based on least privilege <br> - Read basic JSON policy examples | 28/04/2026 | 28/04/2026 | <https://cloudjourney.awsstudygroup.com/1-explore/> |
| Wednesday | - Learn Amazon EC2 <br> - Take notes on AMI, instance type, EBS volume, public IP, Elastic IP, and key pair | 29/04/2026 | 29/04/2026 | EC2 |
| Thursday | - Create a Free Tier EC2 instance <br> - Configure Security Group for SSH/HTTP <br> - Connect to the server using SSH | 30/04/2026 | 30/04/2026 | EC2, Security Group |
| Friday | - Install a basic web server on EC2 <br> - Check browser access <br> - Stop or delete unnecessary resources to avoid extra cost | 01/05/2026 | 01/05/2026 | AWS Console, AWS CLI |

### Week 2 outcomes:

* Understood that IAM is used to manage identity and access permissions in AWS.
* Learned why the root account should not be used for daily work.
* Created a basic EC2 instance and connected to it successfully through SSH.
* Learned how to open required ports using Security Groups and test a simple web application.
* Built the habit of cleaning up resources after practice to control cost.
