During my internship at Amazon Web Services (AWS), I started with very limited knowledge of cloud computing. Therefore, the 12-week internship plan was divided into three main phases: building AWS fundamentals, practicing core AWS services, and applying those services to a movie streaming website project.

The learning and implementation roadmap was based on Cloud Journey - Explore AWS Services, focusing on AWS account setup, cost management, IAM, VPC, EC2, S3, RDS, CloudWatch, Route 53, AWS CLI, and highly available application deployment concepts.

**Week 1:** [Getting familiar with AWS, Cloud Computing, AWS Console, and AWS CLI](1.1-week1/)

**Week 2:** [Learning IAM, account security, and basic Amazon EC2](1.2-week2/)

**Week 3:** [Practicing VPC, Security Group, EC2, and SSH connection](1.3-week3/)

**Week 4:** [Learning Amazon S3, storage permissions, and static website hosting](1.4-week4/)

**Week 5:** [Learning Amazon RDS and database connection for applications](1.5-week5/)

**Week 6:** [Monitoring, cost management, and AWS foundation review](1.6-week6/)

**Week 7:** [Analyzing requirements and designing the AWS architecture for the movie website](1.7-week7/)

**Week 8:** [Developing the movie website frontend and preparing frontend deployment](1.8-week8/)

**Week 9:** [Building backend APIs, authentication, and database connection](1.9-week9/)

**Week 10:** [Integrating Amazon S3 for images, videos, and static assets](1.10-week10/)

**Week 11:** [Testing, monitoring, cost optimization, and system completion](1.11-week11/)

**Week 12:** [Internship summary, report completion, and final presentation](1.12-week12/)

Main reference: <https://cloudjourney.awsstudygroup.com/1-explore/>
