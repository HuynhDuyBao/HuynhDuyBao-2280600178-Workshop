### Week 3 objectives:

* Understand AWS networking fundamentals through Amazon VPC.
* Practice creating subnets, route tables, Internet Gateway, and Security Groups.
* Strengthen EC2 knowledge by deploying a server inside a self-configured VPC.

### Tasks implemented this week:

| Day | Task | Start date | Completion date | Reference |
| --- | --- | --- | --- | --- |
| Monday | - Learn VPC, CIDR block, public subnet, and private subnet <br> - Draw a simple network diagram for a web application | 04/05/2026 | 04/05/2026 | VPC |
| Tuesday | - Create a practice VPC <br> - Create public subnet, route table, and Internet Gateway <br> - Attach an Internet route to the public subnet | 05/05/2026 | 05/05/2026 | <https://cloudjourney.awsstudygroup.com/1-explore/> |
| Wednesday | - Create an EC2 instance in the public subnet <br> - Configure Security Group for SSH and HTTP <br> - Test SSH connection from the local machine | 06/05/2026 | 06/05/2026 | EC2, VPC |
| Thursday | - Practice attaching an additional EBS volume <br> - Format, mount, and verify the volume on EC2 <br> - Take notes on the difference between instance storage and EBS | 07/05/2026 | 07/05/2026 | EBS |
| Friday | - Check the full connection flow from the Internet to EC2 <br> - Capture the main diagram, commands, and configurations <br> - Delete lab resources after completion | 08/05/2026 | 08/05/2026 | AWS Console, AWS CLI |

### Week 3 outcomes:

* Understood the role of VPC in isolating and organizing networks on AWS.
* Created a basic network model including VPC, subnet, route table, and Internet Gateway.
* Deployed an EC2 instance inside a custom subnet and accessed it successfully through SSH/HTTP.
* Learned how to attach an EBS volume and verify storage capacity.
* Gained the ability to read a simple AWS network diagram and explain the access flow to an application.
