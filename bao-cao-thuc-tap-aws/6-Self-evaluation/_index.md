During my internship at **Amazon Web Services (AWS)** from **17/04/2026** to **10/07/2026**, I had the opportunity to experience a real working environment, learn foundational cloud computing knowledge, and gradually apply AWS services to a movie streaming website project.

At the beginning, I had limited AWS knowledge, so the internship focused on building fundamentals through core services such as IAM, EC2, VPC, S3, RDS, CloudWatch, and AWS CLI. After the learning phase, I applied these services to analyze, design, implement, and test a movie website system at an internship demo level.

Regarding work attitude, I tried to complete assigned tasks, document my learning process, communicate when facing difficulties, and gradually improve my self-learning ability, problem analysis, and result presentation.

### Strengths

* Maintained a strong learning attitude, especially when starting from basic AWS knowledge.
* Was able to read documents independently, practice step by step, and document the implementation process.
* Completed the 12-week roadmap from learning fundamentals to applying AWS to the movie website project.
* Understood the importance of security, permissions, and cost control when using AWS.

### Areas for improvement

* Need to become more proactive in teamwork and regular progress communication.
* Need to improve troubleshooting skills, especially for networking, permissions, and application deployment on AWS.
* Need to continue improving cloud architecture design, cost optimization, and system security skills.
* Need to practice clearer technical presentation when reporting project results.

### Conclusion

After the internship, I can see clear progress compared with the beginning. From having limited AWS knowledge, I learned the foundational services, deployed basic resources, integrated them into a practical application, and completed the final report. This is an important foundation for me to continue learning cloud computing, system architecture, and application development on AWS.
